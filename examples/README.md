# CodeMind Examples

This folder contains public-safe starter examples for new users and coding agents.

The examples are intentionally small. They are not full runnable product
projects; they document artifact shape and probe-flow contracts without requiring
Android/iOS SDKs, devices, private credentials, or local runtime evidence. The
live no-device demonstration is generated by `automind smoke offline-demo`.

## Recommended first run: no-device smoke

Run this from the CodeMind checkout:

```bash
./automind.sh smoke offline-demo
```

It creates a local task under:

```text
.automind/tasks/offline_demo_smoke/
```

Then inspect the result:

```bash
./automind.sh status offline_demo_smoke
./automind.sh summary offline_demo_smoke
./automind.sh record-check offline_demo_smoke
```

The static files in [`offline-script-demo/`](offline-script-demo/) explain the artifact shape without bundling local runtime evidence.

## Probe-flow starters

See [`probe-flows/README.md`](probe-flows/README.md).

- [`probe-flows/android-basic.json`](probe-flows/android-basic.json) — starter Android probe-flow structure.
- [`probe-flows/ios-intent-basic.json`](probe-flows/ios-intent-basic.json) — starter iOS action-intent structure.

These are examples of the contract, not templates that should be copied blindly. Always adapt flows to the concrete app, task intent, selectors, risk, authorization, and device state.

## Platform demos

In a full CodeMind checkout, the `demos/` folder contains small Android/iOS demo projects and runners for platform-specific experiments. Some require local platform tools, simulators, or devices. The public skill export may omit these runnable demo projects and keep only the no-device artifact-shape example plus probe-flow starter JSON files.
