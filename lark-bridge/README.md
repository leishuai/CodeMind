# CodeMind 飞书 Bridge

CodeMind 飞书 Bridge 是一个可选入口，让你直接在飞书中与本地 CodeMind
交流和协作。它不会改变 CodeMind 原有的终端、TUI 或 Skill 使用方式。

## 可以做什么

- 在飞书中用自然语言与 CodeMind 交流；
- 将明确的开发需求确认后交给 CodeMind 执行；
- 查看规划、实现、验证等任务进度；
- 通过飞书卡片回答 CodeMind 的待确认问题；
- 在任务执行中追加或修改要求；
- 在同一会话中同时管理多个任务；
- 接收测试结果、关键进度和最终报告；
- 同一台电脑连接一个或多个飞书机器人。

## 使用条件

- 已安装 CodeMind；
- 本机安装 Node.js 18 或更高版本；
- 已安装并登录至少一个 CodeMind 支持的 coding agent，例如 Codex、Claude
  Code 或 Trae；
- 本机保持开机和联网。

首次运行时，CodeMind 会按需构建 Bridge 并引导你完成机器人和工程目录配置。

## 首次连接

运行：

```bash
codemind channel start
```

如果还没有机器人，CodeMind 会引导你：

1. 选择扫码创建新的飞书机器人，或绑定已有机器人；
2. 按提示完成飞书扫码和权限配置；
3. 选择该机器人要处理的工程目录；
4. 在后台启动机器人连接。

自动配置权限失败时，终端会给出手动配置提示。按照提示在飞书开放平台补充权限
后，重新运行启动命令即可。

## 启动机器人

```bash
codemind channel start [botId]
```

- 不传 `botId`：连接所有已注册机器人；已经运行的机器人会被跳过。
- 传入 `botId`：配置或启动指定机器人。

`botId` 是 CodeMind 在本机保存的机器人标识。日常使用通常不需要记忆它，可以在
dashboard 中查看。

启动完成后，机器人在后台运行，关闭当前终端不会中断连接。如果机器人停止，
再次运行 `codemind channel start [botId]` 即可。

## 查看状态

```bash
codemind channel dashboard
```

dashboard 会显示每个机器人当前的：

- 连接状态和进程；
- 绑定的工程目录；
- 活跃任务数量；
- 凭据和用户白名单状态；
- 最近更新时间。

## 在飞书中使用

### 日常聊天

直接给机器人发送自然语言消息即可。普通聊天不会自动创建正式开发任务。
机器人收到消息后会立即添加“正在输入”（Typing）表情，表示消息已经到达
CodeMind。消息处理结束后，无论成功或失败都会移除 Typing，不再显示其它
完成状态表情。

### 创建任务

当你明确提出开发或执行需求时，机器人会先回显需求并请你确认。确认后，
CodeMind 才会创建正式任务并开始规划、实现和验证。

你也可以显式使用：

```text
/ask 修复登录页崩溃并增加回归测试
```

### 查看进度

可以直接询问：

```text
现在进展怎么样？
测试通过了吗？
刚才失败的原因是什么？
```

任务进入新阶段、需要你确认、完成测试或结束时，机器人也会主动推送关键状态。

还可以直接询问计划、实现、验证、失败原因、最终报告、证据或最近日志。机器人
会读取对应任务的真实产物后回答，而不是只根据聊天记录猜测。

### 修改进行中的任务

可以直接发送自然语言要求，例如：

```text
把错误提示也改成中文
补一个登录失败的回归测试
```

机器人会回显要修改的内容，确认后再交给对应任务。

### 管理多个任务

每个任务会分配 `t1`、`t2` 等短码，并在任务消息和卡片中标明。例如：

```text
#t2 改成深色主题
#t1 现在到哪一步了？
```

使用短码可以明确指定目标任务，避免多个任务之间混淆。

### 指定工程目录

机器人首次使用且尚未绑定工程目录时，可以在飞书中发送：

```text
#dir /path/to/project
```

也支持 `#工程`、`#项目` 和 `#目录`。目录必须是运行 CodeMind 的电脑上已经存在
的目录。

## 绑定已有机器人

运行指定机器人启动命令，并在凭据菜单中选择绑定已有机器人：

```bash
codemind channel start <botId>
```

按提示输入飞书应用的 App ID 和 App Secret。凭据只保存在本机。

如果 dashboard 中暂时只显示内部 `botId` 而不是飞书应用名称，按终端提示补充
相应的应用权限即可；这不影响已经配置好的机器人连接。

## 停止机器人

停止指定机器人：

```bash
codemind channel stop <botId>
```

停止所有机器人：

```bash
codemind channel stop
```

## 常见问题

### 飞书中没有收到机器人回复

1. 运行 `codemind channel dashboard` 确认机器人是否已连接；
2. 确认机器人已加入目标会话，且发送人位于允许范围内；
3. 检查飞书开放平台中的消息和事件权限；
4. 查看 `~/.automind/channels/<botId>/daemon.log`；
5. 修正问题后重新运行 `codemind channel start <botId>`。

### 自动配置权限失败

飞书的部分自动配置依赖开放平台当前能力。失败时终端会打印需要补充的权限、
事件和发布步骤。完成手动配置后重新启动机器人。

### 改了工程目录但任务仍使用旧目录

正在运行的机器人会继续使用启动时加载的目录。停止该机器人，再运行
`codemind channel start <botId>` 使新目录生效。已经创建的任务应继续在原工程
目录中完成。

### 机器人意外停止

重新运行：

```bash
codemind channel start <botId>
```

任务状态和已发送进度记录保存在本机，机器人恢复后会继续读取已有任务状态。
